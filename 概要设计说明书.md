# Edge插件“DeepSeek助手”概要设计说明书

**文档版本**：V2.0  
**日期**：2026-07-12  

---

## 1. 引言

### 1.1 编写目的
本文档定义Edge插件“DeepSeek助手”的完整设计方案，说明功能、架构、模块、界面与数据流，用于指导后续开发与测试。

### 1.2 项目背景
用户频繁使用DeepSeek网页版对话，现有痛点：
- 每次需手动输入网址或使用书签进入。
- 重复输入常用指令效率低。

本插件提供工具栏快捷入口、预制化指令集，并实时显示聊天页面状态（可输入/模型输出中），提升效率。

### 1.3 适用范围
- 浏览器：Microsoft Edge（Chromium内核，Manifest V3）
- 目标站点：https://chat.deepseek.com/
- 用户：DeepSeek高频使用者、办公学习人群

---

## 2. 总体设计

### 2.1 设计目标
1. 一键直达对话页面。
2. 预制指令可选，点击填入输入框，可自动发送。
3. 自定义管理指令（增删改查、分类、排序、导入导出）。
4. 实时显示DeepSeek页面状态：空闲可输入 或 模型回复中，输出中禁用自动发送。
5. 轻量无侵入，低资源消耗。

### 2.2 运行环境与权限
- 扩展组成部分：Service Worker（后台）、Popup页面、Options页面、Content Script
- 权限：`storage`、`tabs`、`activeTab`（可选）、`contextMenus`（二期可选）
- Content Script注入范围：仅限 `*://chat.deepseek.com/*`

### 2.3 技术架构（三层 + 状态通道）

```
[ Popup界面 ]  ←→  [ Service Worker ]  ←→  [ Content Script ]
      ↑                    ↑                      (注入deepseek页面)
      |                    |
 [ Options页面 ]  (管理指令)  |
                         [ chrome.storage.local ]
```

- Popup：显示状态、指令列表、搜索、操作按钮。
- Service Worker：协调指令填充、管理标签页、缓存页面状态、代理存储读写。
- Content Script：注入文本、自动发送、监控页面状态并上报。
- Options：独立的指令管理页面。

### 2.4 关键约束
- 目标Edge版本 ≥ 120。
- 数据全部本地存储，不上传云端。
- 不修改DeepSeek页面原有DOM结构，仅操作输入框和模拟事件。
- 状态监听防抖处理，避免性能影响。

## 3. 功能模块设计

### 3.1 模块清单
- M1 工具栏弹窗（Popup）
- M2 后台服务（Service Worker）
- M3 内容注入（Content Script）
- M4 选项管理页面（Options）
- M5 右键菜单（二期可选）

### 3.2 模块详细设计

#### M1 工具栏弹窗（Popup）
- **状态指示条**（新增）
  - 位置：顶部固定，高度28px。
  - 显示内容：`🟢 可输入` 或 `⏳ 模型回复中…`
  - 颜色：空闲浅绿，生成中浅黄，并有省略号动画。
- 指令列表：按分类分组，支持折叠。
- 搜索框：粘性顶部，实时过滤指令标题。
- 点击指令项：发送填充请求，弹窗自动关闭。
- 自动发送开关：指令项右侧有开关图标，当全局状态为“模型回复中”时强制置灰，仅可填入。
- 底部按钮：“打开DeepSeek”、“管理指令”。

#### M2 后台服务（Service Worker）
- 启动时加载存储中的指令配置。
- 核心流程：接收 `EXECUTE_INSTRUCTION` 消息 → 查询/创建DeepSeek标签页 → 等待页面加载完成 → 向Content Script发送 `FILL_TEXT` 消息 → 返回结果。
- **状态管理（新增）**
  - 维护 `tabStatusMap`，记录每个DeepSeek标签页的状态（IDLE 或 GENERATING）。
  - 接收Content Script的 `STATUS_CHANGE` 更新缓存。
  - 标签页关闭或导航离开时自动清除状态。
  - 响应Popup的 `GET_STATUS` 请求，返回当前活跃标签页状态。
- 提供数据操作：`GET_INSTRUCTIONS`、`SAVE_INSTRUCTION`、`DELETE_INSTRUCTION` 等。

#### M3 内容注入（Content Script）
- **指令注入子模块**
  - 定位输入框：主选择器 `textarea[placeholder*="消息"]`，备选多个特征选择器。
  - 填入文本并触发 `input` 事件。
  - 自动发送：仅在页面状态为IDLE时，模拟点击发送按钮或触发Enter键；若状态为GENERATING，只填充并返回警告。
- **状态监控子模块（新增）**
  - 使用 `MutationObserver` 持续监听DOM变化，判定规则：
    - 存在“停止生成”按钮 → GENERATING
    - 输入框被禁用且无停止按钮 → GENERATING
    - 其他情况 → IDLE
  - 状态变化时防抖500ms，通过 `STATUS_CHANGE` 上报Service Worker。
  - 页面卸载时发送 `DISCONNECT`。
  - 若观测元素丢失，默认上报IDLE以保证功能可用。

#### M4 选项管理页面（Options）
- 左侧分类树（全部指令、各分类），右侧表格列表。
- 指令编辑：对话框或内联编辑，支持拖拽排序。
- 功能：增删改、批量启用/禁用、导入导出JSON、恢复默认指令包。
- 每条指令包含：标题、内容、分类、是否自动发送、排序权重、是否启用。

#### M5 右键菜单（二期）
- 选中文本后右键菜单可选“发送至DeepSeek”或指定指令。
- 菜单项可在选项中配置。

## 4. 界面设计要点

### 4.1 Popup 弹窗
- 尺寸：宽320px，高自适应最大480px。
- 布局顺序：状态指示条 → 搜索框 → 可滚动指令列表 → 底部操作栏。
- 指令项：左侧图标（可选），标题，右侧自动发送开关（智能禁用）。
- 支持暗色模式跟随系统。

### 4.2 Options 页面
- 风格简洁，与Edge设置类似。
- 左侧分类树可添加、重命名分类。
- 右侧表格显示标题、内容预览、分类、自动发送、拖拽把手、操作按钮。
- 右上角工具栏：导入、导出、恢复默认。

### 4.3 图标与徽章（V1.1增强）
- 工具栏图标默认DeepSeek风格。
- 当有标签页处于GENERATING状态时，图标上加橙色圆点，悬停提示“模型回复中”；IDLE时无徽章。

## 5. 数据设计

### 5.1 存储方式
使用 `chrome.storage.local`，存储键：
- `instructions`：指令数组
- `settings`：`{ defaultAutoSend: false }`

### 5.2 指令数据结构
```json
{
  "id": "唯一UUID",
  "category": "通用",
  "title": "总结内容",
  "text": "请对以下内容进行要点总结：",
  "autoSend": false,
  "enabled": true,
  "order": 1,
  "showInContextMenu": false
}
```

默认内置10条常用指令（涵盖通用、语言、编程、写作、办公等）。

### 5.3 状态数据流（新增）
- Content Script检测状态变化 → 上报 `STATUS_CHANGE` {status, tabId}
- Service Worker更新缓存
- Popup打开时请求 `GET_STATUS` → 获取当前状态并显示
- 状态影响：输出中禁止自动发送，弹窗按钮置灰。

## 6. 接口与通信设计

### 6.1 通用消息格式
请求：`{ "type": "消息类型", "payload": {...} }`  
响应：`{ "success": true, "data": {...}, "error": null }`

### 6.2 消息列表

| 类型 | 方向 | 说明 | 负载示例 |
|------|------|------|----------|
| `EXECUTE_INSTRUCTION` | Popup → SW | 执行某条指令 | `{ instructionId }` |
| `FILL_TEXT` | SW → Content | 填入文本到输入框 | `{ text, autoSend }` |
| `FILL_RESULT` | Content → SW | 填充/发送结果 | `{ status, warning }` |
| `GET_INSTRUCTIONS` | Popup/Options → SW | 获取全部指令 | `{}` |
| `SAVE_INSTRUCTION` | Options → SW | 保存/更新指令 | `{ instruction }` |
| `DELETE_INSTRUCTION` | Options → SW | 删除指令 | `{ id }` |
| `STATUS_CHANGE` (新) | Content → SW | 页面状态变更 | `{ status, tabId }` |
| `GET_STATUS` (新) | Popup → SW | 请求当前状态 | `{}` |
| `STATUS` (新) | SW → Popup | 返回状态 | `{ status, tabId }` |
| `DISCONNECT` (新) | Content → SW | 页面卸载通知 | `{ tabId }` |

## 7. 异常处理与容错

- **标签页未打开**：新建并等待加载，超时5秒提示。
- **输入框定位失败**：尝试多个备选选择器，仍失败则复制指令到剪贴板并提示。
- **自动发送时状态为GENERATING**：只填充不发送，返回警告。
- **状态监听失效**：默认上报IDLE，Popup显示“状态未知”但所有按钮可用。
- **存储异常**：使用空数组默认值，保证插件启动。
- **性能保护**：状态上报防抖500ms；Popup关闭后不轮询。

## 8. 版本规划

- **V1.0**：基本弹窗、指令填入、状态显示（弹窗内）、指令管理、默认指令包。
- **V1.1**：自动发送与状态联动、搜索、图标徽章、导入导出。
- **V2.0**：右键菜单、快捷键、云端同步（可选）、多语言。

---

## 9. 验收标准

- 弹窗加载 < 200ms，指令填入 < 2秒（新标签页<5秒）。
- 状态变化到弹窗显示 < 1秒。
- 指令增删改即时生效，无需刷新。
- 无网络或站点不可用时友好提示。
- 状态监控内存占用低，不引起页面卡顿。
- 顺利通过Edge扩展商店审核。
